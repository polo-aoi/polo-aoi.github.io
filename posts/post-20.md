# 我用 Claude 给自己写了一个博客编辑器

## Publii？不了，我要的是我的编辑器

昨天在 Gemini 的建议下，我下载了 Publii。

一个号称"开源、本地化、所见即所得"的 CMS。听起来完美。安装，启动，看着满屏的配置项和模板选项，我沉默了。

我不需要邮件订阅。不需要 SEO 元数据面板。不需要多语言切换。不需要会员系统。我就想安安静静写篇文章，点个按钮，推到我的 GitHub Pages 上。

Publii 是辆房车，而我只想要一辆自行车。

关掉。打开 Claude。

---

## 直接 Vibe Coding

什么是 Vibe Coding？就是脑子里有画面，嘴巴能说出来，AI 帮你把画面变成代码。

我跟 Claude 说了我的需求：

- 本地跑一个编辑器，浏览器里写 Markdown
- 左边是文章列表，右边是编辑区，和 GitHub README 编辑器一样的味道
- 文章按分类组织，主页用手风琴折叠
- 图片拖进去自动压缩上传
- 写完一键推送到 GitHub Pages
- 换设备了 clone 下来就能继续写

没有 PRD，没有技术选型会。就是聊着天，看着代码一行一行长出来。

---

## 这个编辑器的样子

### 写文章

侧边栏点「写新文章」，右边出来编辑界面。标题、分类、标签、封面，都在顺手的位置。正文写 Markdown，切到 Preview 实时看效果。

图片拖进去就行，自动压缩到 1600px、转 JPEG、85% 质量。粘贴剪贴板的截图也可以。上传完还能一键设成文章封面。

自动保存到本地，1.5 秒防抖。写一半关浏览器也不怕。

### 管理分类

手风琴折叠的主页有五个分类：开发日志、技术文章、摄影设计、生活随笔、水处理。编辑器里拖拽排序，分类改名，增减分类，都行。

### 发布推送

写完点「发布并推送」——草稿变已发布，内容自动 commit + push 到 GitHub。侧边栏底部还有一个独立的「同步推送」，删文章、改分类这类操作完，点一下就同步。

### 整个技术栈

Flask + 原生 HTML/CSS/JS，零前端框架。Markdown 用 marked.js 渲染。数据全存文件（data.json + posts/\*.md），Git 仓库就是数据库。编辑器代码也在同一个仓库里，换台电脑 clone 下来 `python3 server.py` 完事。

## 想清楚了再写代码

昨天的 Publii 到今天的自定义编辑器，最深的感受是：**工具应该是你思维的外延，不是你思维的枷锁**。

Publii 没有问题，它有它的用户。但 Vibe Coding 的魔力在于：你可以绕过所有"通用方案"，直接抵达自己想要的那个东西。不需要妥协，不需要学习一套你不关心的系统，不需要在几百个配置项里找那个你需要的。

你要什么，你就说。AI 帮你写。

这才是编码该有的样子。
